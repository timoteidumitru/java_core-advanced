Option                                               Purpose
-XX:+UseZGC                                           Enables the Z Garbage Collector.
-XX:SoftMaxHeapSize=<size>                            Sets preferred soft heap limit for dynamic memory control.
-XX:+ZUncommit                                        Allows unused memory to be returned to the OS.
-Xms / -Xmx                                           Set initial and maximum heap size.
-Xlog:gc*                                             View GC activity (Java 9+).

Run Command:
java -XX:+UseZGC `
     -Xms25g -Xmx25g `
     -XX:SoftMaxHeapSize=25g `
     -XX:+ZUncommit `
     -Xlog:gc*,gc+stats=trace:file=src/main/resources/IO_operations/gc-perf.log `
     -cp target\classes `
     com.javaPlayground.garbageCollectors.ZGC.CsvReportAnalyzerZGC_Heavy

Summary:
ZGC is ideal for large-scale, low-latency applications that demand predictable performance under high memory load.
It achieves this through concurrent marking, relocation, and remapping phases, ensuring that the Java application
threads continue executing with minimal interruption.
